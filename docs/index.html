The problem & the fix

Decks that break right before you present.

The problem

A Claude Design deck export needs a server — and the internet — just to open.

Decks exported as *.dc.html normally need a local web server and a live internet connection (Google Fonts) just to present, and they have no speaker notes or presenter view. Awkward right before a talk or interview.

What Snapdeck does

One command produces a folder you double-click to present.

Snapdeck converts that export into a self-contained <name>-offline/ folder — fonts, images, GIFs and the deck engine all bundled in, with zero network requests at play time.

$ uv run snapdeck.py your-deck.dc.html
Features

Everything bundled, nothing left to chance.

From a single command to laser pointers — Snapdeck packs a full presenting kit into one offline folder.

One command

Convert any deck in a single line: uv run snapdeck.py your-deck.dc.html.

Fully offline

Google Fonts mirrored locally and referenced images & GIFs copied in — zero external requests at play time.

Full-bleed playback

Edge-to-edge playback page with keyboard navigation — arrows, Space, digits — plus a “▶ Fullscreen” button.

Presenter view

A dedicated presenter.html shows the current slide, the next slide, your speaker notes and a timer — kept in sync with the audience window over BroadcastChannel.

On-slide annotation

Laser pointer, pen (red, yellow, blue), eraser, undo and clear. The toolbar auto-hides when idle and is fully draggable.

Animation replay

Per-slide entrance and looping animations play when you arrive on a slide — not silently at load.

One-click macOS launcher

A 双击放映.command launcher starts a local offline server and opens the deck — with a portable serve.sh too.

Faithful rendering

Template-driven slides are rendered once in headless Chrome and the finished DOM is snapshotted — so nothing is ever bundled half-empty.

Quick start

From export to presenting in three lines.

1

Install uv once

A single installer sets up uv, which manages Python and the one dependency.

2

Convert your deck

Point Snapdeck at your .dc.html export and it builds the offline folder.

3

Double-click to present

Open the macOS launcher inside the new -offline/ folder and you’re live — no internet needed.

terminal 
# install uv once:  curl -LsSf https://astral.sh/uv/install.sh | sh
$ uv run snapdeck.py your-deck.dc.html
$ open your-deck-offline/双击放映.command
How it works

Render once. Snapshot. Bundle. Replay.

Render the live deck once with its runtime, snapshot the finished DOM, localize fonts and assets, bundle the deck engine, and replay animations on each slide. The result is a folder that presents anywhere, offline.

01

Render live

Run the deck once in headless Chrome with its real runtime.

02

Snapshot DOM

Capture the finished DOM so nothing renders half-empty.

03

Localize assets

Mirror fonts and copy in images & GIFs locally.

04

Bundle engine

Pack the deck engine into the self-contained folder.

05

Replay animations

Play entrance & looping animations on each slide.

The result: a folder that presents anywhere, offline.

Requirements

Just two tools — and the internet only at convert time.

uv

Manages Python plus the single dependency, lxml — nothing else to install by hand.

Google Chrome / Chromium

Used for the headless render pass that snapshots each finished slide.

Internet at convert time only

A connection is needed once, while converting — and never again when you present.
